Převod textu na hlas — ChatterBox vs OmniVoice

Dva open-source modely pro zero-shot voice cloning, dvaadvacet českých hereckých hlasů, jedna a tatáž věta na každý hlas. Vlevo je referenční nahrávka, vpravo dva výstupy — jeden od ChatterBoxu, druhý od OmniVoice. Stejný vstupní text, stejná referenční ukázka, slyšte rozdíl ve výslovnosti, barvě hlasu i přízvuku.

ChatterBox

Resemble AI · MIT · čeština přes lokální T3 fine-tune

ChatterBox je open-source TTS model od Resemble AI s podporou zero-shot voice cloningu — z krátkého referenčního klipu (10–20 s) napodobí daný hlas a přečte libovolný text. Base model je multilingvální, ale pro českou výslovnost používáme vlastní lokální T3 fine-tune (t3_cs.safetensors, ~2,1 GB), který natrénoval autor stránky. Tahle pipeline pohání všechny audio povídky na našem YouTube kanálu.

Setup

  • HW: NVIDIA RTX 3060 (8 GB VRAM). Base model ~3,3 GB, syntéza dalších ~3 GB.
  • Model: base ResembleAI/chatterbox + lokální český T3 fine-tune.
  • Server: tts_server.py (FastAPI) běží jako systemd service, drží model v paměti, poslouchá na 127.0.0.1:8767. Endpoint POST /tts přijímá { text, reference_voice, exaggeration, cfg_weight }.
  • Knihovna hlasů: jeden referenční WAV na hlas (16 kHz mono, normalizováno na −22 dBFS RMS).

Jak se generuje ukázka

POST /tts { text: "Dobrý den, jmenuji se Miroslav Moravec...",
            reference_voice: "voices/moravec.wav",
            exaggeration: 0.45 }

Pipeline pro audio povídky

Tato pipeline pohání všechny audio povídky na našem YouTube kanálu. Scénář se rozdělí po replikách, každé je přiřazen jeden hlas, vše projede TTS serverem do MP3 a obalí se vizuálem v 1080p H.264. Příklad hotové povídky: „Léto v Australisu“ (~45 min, 12 hlasů, audio + AI obrázky + video).

Zdroje

Zdrojový kód modelu: github.com/resemble-ai/chatterbox. Base váhy: huggingface.co/ResembleAI/chatterbox. Český T3 fine-tune trénovaný lokálně, není veřejně publikovaný.

OmniVoice

k2-fsa · Apache-2.0 · 600+ jazyků zero-shot bez fine-tune

OmniVoice je multilingvální TTS model od k2-fsa, postavený nad Qwen3-0.6B s diffusion-LM architekturou. Podporuje přes 600 jazyků (včetně češtiny) zero-shot, takže pro češtinu nepotřebuje žádný jazykový fine-tune — funguje rovnou z public váh. Klonování probíhá jako few-shot u LLM: reference se zakóduje na tokeny pomocí audio_tokenizer.encode() a model dostane [ref_audio_tokens + ref_text + target_text] jako jeden prompt.

Setup

  • HW: NVIDIA RTX 3060, ~4 GB VRAM (váhy + inference). Funguje i na CPU/Apple Silicon.
  • Model: k2-fsa/OmniVoice — staženo jednou (~2 GB do ~/.cache/huggingface/), dál běží lokálně.
  • Referenční délka: stačí 3–10 s, model si s delším klipem poradí sám. Pro lepší výsledky doporučuje fixní ref_text — přepis se získává Whisperem s explicitním language="czech".
  • Rychlost: RTF ~0,025 (40× rychlejší než realtime) — generování jedné věty ~3 s, ideální pro reálnou interakci.

Jak se generuje ukázka

prompt = model.create_voice_clone_prompt(
    ref_audio="reference/cr-voices/moravec.wav",
    ref_text="Nejraději chodil Hugo s Jirkou na ostrůvek...")
audio  = model.generate(text="Dobrý den, jmenuji se Miroslav Moravec...",
                        voice_clone_prompt=prompt)

Více informací

Kompletní instrukce, příklady (voice design, voice cloning, batch zpracování) a hromadné klonování všech 22 českých hereckých hlasů najdete v repu Olbrasoft/OmniVoiceExample. OmniVoice na Hugging Face: k2-fsa/OmniVoice, online demo: spaces/k2-fsa/OmniVoice.

Použití pro výzkumné a vzdělávací účely. Autoři OmniVoice výslovně zakazují použití pro neautorizované klonování hlasu, podvody a impersonaci.
